Java 注解

理解

  1. Annotation 也被称为元数据（Metadata），用于修饰包、类、方法、属性、局部变量等数据信息；
  2. 和注释一样，注解不影响程序逻辑，但注解 可以被编译或运行，相当于嵌入在代码中的补充信息；

元注解

  1. @Retention：指定注解三种作用范围

    1. RetentionPolicy.SOURCE：编译器使用后，直接丢弃这种策略的注释；
    2. RetentionPolicy.CLASS（默认值）：编译器将把注解记录在 class 文件，当运行 Java 程序时，JVM 不会保留注解；
    3. RetentionPolicy.RUNTIME：编译器将把注解记录在 class 文件，当运行 Java 程序时，JVM 会保留注解，程序可以通过反射获取该注解；

  2. @Target：指定被修饰注解能作用于哪些程序元素上，包含 value 的成员变量

  3. @Documented：用于指定该元注解修饰的 Annotation 类将被 javadoc 工具提取成文档，即在生成文档时，可以看到该注解。

  4. @Inherited：被它修饰的 Annotation 将具有继承性，如果某个类使用了被 @Inherited 修饰的注解，则其子类自动具有该注解。
